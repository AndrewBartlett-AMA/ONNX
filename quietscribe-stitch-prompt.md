# Quiet Scribe - Stitch / UI Generation Prompt

Use the following prompt in Stitch or any UI-generation tool.

## Prompt

Design a premium Progressive Web App called **Quiet Scribe**.

Quiet Scribe is a private, local-first speech-to-text transcription app that runs primarily in the browser and can be installed as a desktop-style PWA. The interface should feel polished, minimal, premium, and highly usable, with a strong focus on clarity and privacy.

### Product characteristics

- installable PWA
- offline-capable app shell
- speech-to-text transcription
- microphone recording and audio file upload
- transcript editing workspace
- local save history
- export/download transcript
- technical diagnostics available but not visually dominant
- modern AI utility, but calm and restrained

### Visual direction

Create a clean, elegant interface inspired by premium productivity software.

Use:

- generous whitespace
- rounded panels
- subtle shadows
- clean typography
- strong hierarchy
- a calm, refined accent color
- both light mode and dark mode concepts

Avoid:

- clutter
- gimmicky AI visuals
- neon sci-fi styling
- overly complex dashboard layouts

### Key screens to design

1. **Home / app shell**
   - app title and short privacy-first message
   - prominent start recording button
   - upload audio button
   - install app prompt
   - recent transcripts list
   - backend status badge

2. **Recording screen**
   - large microphone control
   - timer
   - waveform or input activity visualization
   - pause, stop, cancel actions
   - subtle backend/model indicators

3. **Transcript workspace**
   - editable transcript area
   - transcript title
   - status bar showing save state, backend, duration, and model
   - export menu
   - polished loading and processing states

4. **History screen**
   - searchable local transcript list
   - card or row layout
   - metadata such as date, duration, and source
   - actions for open, export, delete

5. **Settings / diagnostics panel**
   - backend preference
   - model selection
   - browser capability checks
   - local storage and cache status
   - clean advanced-settings look

### Experience goals

The app should feel:

- private
- fast
- native-like
- installable
- calm
- trustworthy
- premium

### Layout guidance

Desktop should use a clean two-panel layout with navigation/actions on the left and the main transcript workspace on the right.

Tablet and mobile versions should preserve clarity with stacked cards and sticky key actions.

### Design keywords

premium productivity app, local-first AI, private transcription workspace, elegant minimalism, browser-native desktop utility, polished PWA, calm intelligence
